# 通用代码规范与质量审计专家 Skill

本 Skill 旨在将现代软件工程中最核心、跨语言的最佳实践与编码规约（命名、异常资源管理、并发控制、数据库设计、安全漏洞、架构原则）提炼为一套通用的审计机制，支持 Agent 或开发者在任意多语言环境下直接调用。

## 运行环境要求

- 浏览器环境：不限制（可分析页面上的任意代码编辑器或网页文本）
- 沙箱环境：推荐（用于运行内置静态扫描器对多种语言文件做匹配，以及编辑重构本地文件）
- 用户本地目录授权：可选（若需批量审查本地工程代码）

## 能力路由机制

- **多语言自适应审查**：根据输入的源代码或文件扩展名，自适应匹配对应语言的最佳规约集。
- **通用设计原则咨询**：对于复杂的架构设计，提供基于经典 SOLID 原则、防御性编程以及高性能并发的评估。
- **正反例自适应生成**：不仅指出问题，还可以针对当前的编程语言（Java, Python, Go, C++, JS/TS 等）生成定制的重构正反例。

## 核心工作流

1. **识别语言与环境**：当触发本 Skill 时，首先检测输入的代码语言（或文件后缀），并加载对应的规约分支。
2. **规范匹配与扫描**：对照 `references/guidelines.md` 中的【通用强制级】、【语言特定推荐级】以及【架构参考级】规范，对代码进行合规性过滤。
3. **隐患评估**：标出不合规的代码位置、属于什么规约，并评估其技术危害（如：竞争条件、OOM、资源泄漏、SQL 注入、反序列化异常等）。
4. **生成重构正反例**：针对该语言提供直观的代码对比：
   - 坏味道代码（反例）。
   - 符合规约的代码（正例）。
   - 底层原理深度解析（“知其然，更知其所以然”）。
5. **代码编辑回写**：如在沙箱中，使用 `e2b_edit` 或脚本工具将重构后的代码无缝回写，并运行简单语法检测。
6. **输出多语言审计报告**：提供结构化的 Markdown 审查文档。

## 关联文件

- `SKILL.md`：Skill 的元信息、工作流和路由。
- `references/guidelines.md`：涵盖多种主流语言的核心规约精简版字典（命名、资源、并发、异常、数据库、安全、SOLID 原则）。
- `scripts/generic_scanner.py`：一个可直接运行的、基于正则表达式的多语言通用代码坏味道静态检测脚本。

## 交付契约 (Output Contract)

每次执行规约审查，必须提供如下结构的 Markdown 报告：

```markdown
### 通用代码规范审计报告

#### 一、 审计概览
- **审查目标**：[如：UserService.py / config.go / index.js]
- **目标语言**：[Python / Go / TypeScript / Java / C++ 等]
- **问题汇总**：
  - 🔴 【强制】违反：X 处 (高危：如资源泄漏、SQL注入、并发竞争)
  - 🟡 【推荐】违反：Y 处 (中危：如命名混淆、代码坏味道、硬编码)
  - ⚪ 【参考】违反：Z 处 (低危：如注释缺失、可以更优雅的重构)

#### 二、 详细违规项与整改建议
1. **[级别] 规约条目名称（针对 [具体语言]）**
   - **违规代码**：`[代码片段]`
   - **危害分析**：[详细描述其隐患，如：导致 goroutine 泄漏/连接池耗尽/内存溢出]
   - **重构建议**：[如何修改]
   - **正例展示**：
     ```[language]
     // 规范的正例
     ```

#### 三、 最终重构完成代码
[提供完全符合规约的、可以直接替换使用的完整类/函数代码]
```

## 异常与容错处理

- **语言无法识别**：若输入的是伪代码或极小片段，默认退化为通用编程规范（如变量命名清晰、防空指针、合理异常捕获、资源释放）进行匹配。
- **文件包体积过大**：若需对多级目录进行全量扫描，禁止一次性将所有文件读入上下文。应在沙箱中通过 `scripts/generic_scanner.py` 生成预检 JSON 文件，再针对违规严重的重点文件进行精细化 Review。
